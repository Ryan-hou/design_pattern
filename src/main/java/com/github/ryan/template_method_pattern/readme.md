#### 模版方法模式
在一个方法中定义算法骨架,而将一些步骤延迟到子类中.模版方法使得子类可以在不改变算法结构的情况下,重新定义算法中的某些步骤

好莱坞原则: don't call me, we'll call you

好莱坞原则给我们提供了一种防止"依赖腐败"的方法,在好莱坞原则下,我们允许低层组件将自己挂钩到系统上,但是高层组件会决定什么
时候和怎么使用这些低层组件.换句话说,高层组件对待低层组件的方式是"别调用我们,我们会调用你"

####jdk中的应用
* java.io 的 InputStream 类有一个 read() 方法,是由子类实现的,
而这个方法又会被 read(byte[] b, int off, int len)模版方法使用
* Java Arrays类 的排序方法也提供了一种模版方法模式的变式:
在 mergeSort 模版方法中,依赖于 comparable 接口的 compareTo 方法来完成算法
